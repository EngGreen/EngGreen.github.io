프로그래밍 개론

CPU, RAM, SSD

  • CPU
    • CPU는 두뇌(계산기)의 역할을 한다.
    • 하지만, 기억력이 짧다.
  • RAM (=데이터 저장소)
    • 전원이 꺼지면 데이터가 소실된다. (=휘발성 메모리)
  • SSD (=반영구 기억장치)

※ 리그 오브 레전드의 실행과정

  1. 리그 오브 레전드의 데이터를 SSD에 저장한다.
  2. 바탕화면에서 리그 오브 레전드를 실행한다.
  3. SSD에서 실행에 필요한 필수적인 내용을 RAM으로 이동시킨다.
  4. RAM에서 이동시킨 내용 중 실행에 필요한 부분을 CPU로 이동시킨다.

이 때 SSD에서 RAM으로 이동시킨 내용은 CODE(TEXT), DATA, HEAP, STACK의 네 부분으로 나눌 수 있다.
이 중 CODE(TEXT) 부분에는 CPU가 해야하는 일이 포함되어 있고, 이 부분을 CPU로 전달한다.
그 후 CPU 에서 디코드(분석)한 후 실행한다.

여기서 CPU, RAM, SSD등에 저장되는 정보는 ‘1010111100’과 같은 이진법의 컴퓨터 언어로 작성되어 있다.
이런 이진법의 컴퓨터 언어를 인간이 이해하고 프로그래밍하는것은, 사실상 불가능하다.

그러므로 영어로 cout << hello; 와 같은 문장을 타이핑 하고, 이것을 이진수로 번역해 컴퓨터가 이해할 수 있도록 돕는다.

컴파일러에 작성할 수 있는 문법은 다양하며, 문법에 따른 언어들을 프로그래밍 언어라고 부른다.

이 때 번역하는 과정을 컴파일이라고 부르며, 번역해주는 도구를 컴파일러라고 부른다.

Visual Studio는 코드 작성 및 컴파일을 할 수 있게 해주는 프로그램이다.

프로그래밍 언어 별 차이 예시 <Python vs C++>

  • Python
    • 문법이 간단하고 쉽다.
    • 언어차원에서 많은 부분이 내포되어있고 관리해준다.
    • 주로 웹, 툴 개발에 사용된다.
  • C++
    • 문법 작성이 어렵고 복잡하다.
    • 여러 부분에서 직접 제어하고 관리가 가능해, 효율이 더 좋고 빠르다.
    • 주로 예민한 부분인 게임 개발, 운영체제, 인공지능의 코어부분 등의 개발에 사용된다.
변수의 기초

램 내부에 가상의 공간을 ‘변수‘라고 부르며, 공간을 할당하는 행위를 ‘변수선언‘이라고 부른다.

변수 선언의 구조 = [타입] [이름];

ex) int hp; = hp라는 변수를 int라는 타입으로 설정한다는 의미.

※ 끝에오는 세미콜론(;)은 C++에서 한 문장이 끝났다는 뜻이다.

데이터 형식 범위

데이터 형식의 범위가 정리되어있는 페이지

대표적으로, 정수를 나타내는 데이터 타입

  • Char (1바이트) / -128 ~ 127
  • short (2바이트) / -32,768 ~ 32,767
  • int (4바이트) / -2,147,483,648 ~ 2,147,483,647
  • long long (8바이트) / -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807

위의 타입 앞에 unsigned를 붙이면 범위에서 음수가 사라지고 양수의 범위가 x2+1까지 늘어난다.

→ unsigned car 의 범위는 0~255

예를 들어, hp를 설정하는 경우 int (4바이트)정도만 사용하면 되므로 설계 전 어느 정도를 사용할지 생각이 필요하다.

이렇게 RAM에 저장되어있는 정보를 불러와 콘솔에 출력하는 과정을 ‘콘솔 아웃(console out)‘이라 한다.

콘솔 아웃의 구조 = cout << [이름];

cout이라는 명령어에 대해서는 추후 객체지향 부분에서 다시 심도있게 공부한다.

앞의 내용을 정리해서, maxHp가 200인 상황에서 포션, 부활 등으로 hp가 maxHp가 됐을 때의 hp를 출력하는 코드는 다음과 같다.

int hp;                 //hp라는 변수를 설정
int maxHp;              //maxHp라는 변수를 설정

int main()
{
    maxHp = 200;        //maxHp의 값 = 200
    
    hp = maxHp;         //hp는 maxHp와 같음

    cout << hp;         //hp의 값을 콘솔에 출력
}

여기서 주의해야 할 부분은 hp를 직접 200으로 설정하면 추후에 maxHp가 늘어났을 경우 대응이 불가능해진다.

Visual Studio에 이렇게 입력한 후, 빌드를 해보면 200이 출력된다.

정수의 원리

데이터의 최소 단위는 0혹은 1의 표시가 가능한 ‘비트(bit)‘이다.

이 비트가 8개로 이루어진, 즉, 1001 1101과 같은 데이터를 ‘바이트(byte)‘라고 부른다.

→ mp라는 변수를 short로 설정 후, short에 306,189,112라는 값을 설정한다.

그리고 빌드를 해보면 4,920이라는 엉뚱한 수가 나온다.

∵ 306,189,112라는 4비트만큼의 크기를 가진 수를 설정한 타입인 2비트의 short로 출력한 결과값이 4,920이기 때문이다.

즉,

short mp;

int main()
{
    mp = 30000;
    
    mp += 20000;
    
    cout << mp;
}

를 출력하면 50000이 나와야 하지만, short의 범위를 벗어나기 때문에 엉뚱한 수가 나타나고 이것을 ‘정수 오버플로우‘라 부른다.

진법

  • HEX(16진법) 2,488 / 접두어 = 0x
  • DEC(10진법) 9,352 → 인간 친화
  • OCT(8진법) 22,210 / 접두어 = 0o
  • BIN(2진법) 0010 0100 1000 1000 → 컴퓨터 친화 / 접두어 = 0b

→ hp = 0b1100을 입력후 출력한 결과값 = 12

16진법은 2진법과 궁합이 좋다.

∵
→ 0x 2,488
→ 0b 0010(2) 0100(4) 1000(8) 1000(8)

즉, 2진법으로 나타낸 수를 4자리마다 끊어 16진법으로 표현하면, 같은 수가 된다.

변수 마무리

문자

char ch;

int main()
{
    ch = 120;
    
    cout << ch;
}

를 출력하면 120이 나와야 하지만, x라는 문자가 출력되는 걸 볼 수 있다.

∵ char 자료형으로 선언한 변수에는, 그 숫자가 담기는 것이 아니라 그 숫자에 해당하는 아스키 코드 값이 담기기 때문이다.

소수

float(4바이트) / 3.4E+/-38(7개의 자릿수)
double(8바이트) / 1.7E+/-308(15개의 자릿수)

※ float의 값에는 float hp = 3.43f와 같이 값 뒤에 f를 붙여야 float로 인식한다.

참, 거짓

bool은 참, 거짓을 나타내는 데이터 타입이다.

true와 false를 판단해, 1(true)과 0(false)로 표현한다.

논리적인 값을 표현하는데 사용한다.

bool playSound;

int main()
{
    playSound = false;

    playSound = true;
}

와 같이 사용한다.
